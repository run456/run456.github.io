使用 Gridea + Github 搭建个人博客

2021-03-05 4 min read # blog # 教程 # Gridea
⭐️ Github博客快速搭建指南。

一、建立自己的个人博客

[1] 为什么要使用博客

个人博客面向的对象首先是自己，人的记忆力有限，当前掌握的技能或者知识，随着时间的推移而变得模糊。这就需要将当前获取到的信息记录下来，无论是技术、感悟还是阅读。

与笔记不同的是，博客是需要发表的，无论是面向自己还是他人，都要求把零散的笔记给系统化，这将有助于提升我们的表达能力以及对知识的理解，有些类似费曼学习法，通过用自己的语言将信息重组，来获得更好的知识吸收。

顺便满足自己的码字欲。

[2] 博客平台的选择

在使用过LOFTER、简书这类轻blog之后，基于对国内服务商的不信任（关键字和谐、服务不稳定等因素），需要一个更好友好、开放的平台，最终还是决定使用Github Page作为个人博客。

在一月初有这个想法之后，使用Hexo框架进行建站，光是研究主题的layout就花费了不少时间，美化完web页面后，发现选用的主题对移动端不太友好，而且比较厚重，遂放弃。

在浪费了几天的之后，@Alano 推荐了Hugo框架以及一个较为清爽的主题，又是新一轮的折腾。生命不止，折腾补休，在折腾道路上远比码字来的费时间。

最终敲定了使用Gridea框架。
1.搭建简单，不需要繁琐的git命令，极大的提高了效率。
2.自带的markdown编辑器更符合国人的习惯。
3.主题设计很清新。

至少在未来一年内可以安心码字了。🙄

二、实战操作

[1] 快速搭建Github博客

使用Gridea来搭建博客非常简单，不需要繁琐的命令行以及配置。5分钟内就能搭建好自己的博客。参见 官方文档 。

注意事项：
1.在同步博客时，有时会遇到无法同步的情况，基本上都是由于无法连接github服务器引起的，建议挂ss全局代理。

[2] 设置博客自定义域名

以dnspod为例。
1.在域名解析中，增加一条CNAME类型的记录值，记录值为github page地址。

2.在github的仓库设置中，添加自定义域名，保存即可。

[3] 加速博客访问速度

鉴于Github在大陆访问速度较慢，需要使用Vercel对其进行加速处理。

Vercel 是什么?

Vercel (之前也叫 Zeit 或 now.sh) 是一家提供静态网站托管的云平台，支持从 Github、GitLab等代码仓库中自动拉取代码，进行项目打包和部署等功能。

使用Github账号登录Vercel，选择需要导入的仓库，完成。

导入并自动配置完成后，将会生成一个 xxx.vercel.app 的域名，并支持https访问。经过Vercel优化后，极大的提高了国内访问blog的速度。

在Vercel中绑定自定义域名：
1.如果有在 Github 中绑定过自定义域名，需要先将其删除。
2.在 Vercel 的 Project Settings 中添加域名，并将 dnspod 里的解析值改成Vercel提供的网址，等待解析完成即可。

至此 Gridea + Github 搭建的个人博客优化完成。
Enjoy it! 😘

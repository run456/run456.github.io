使用Github自建免费图床

2021-03-05 3 min read # blog # 教程 # Gridea
🤔如何优雅在的 Markdown 中插入图片。

一、前言

开始写博客后，接触了 Markdown 写作，最头疼的是图片的插入。一开始使用 typora 编辑器，使用本地绝对路径的方式调取图片，但生成静态网页后，图片一直无法正常显示。

因此图床是绕不看的话题，国内外都有不少提供图床服务的网站，但始终不放心，因此还是利用Github自建免费的图床，对于个人博客来说，Github完全够用了。

操作流程简单：新建仓库——上传图片——调用图片Url——完成。

但是为了更快、更方便的使用，需要配合PicGO + jsDelivr使用。

二、设置PicGo

1.新建Github仓库，并在账户设置（ Settings > Developer settings > Personal access tokens ）中创建一个仓库 Token，勾选上 repo 的相关权限即可，获取到Token值。

2.下载PicGo，填写相关信息。

设置好之后就可以快速、便捷的进行图片调用了。

  • 自定义域名可供CDN加速使用。
  • 由于Github服务器的不稳定性，建议在上传图片时，设置ss全局翻墙。

三、为图床加速

创建Github图床，可以使用jsDelivr 对其进行CDN加速，提高国内访问的速度。

jsDelivr是什么？

jsDelivr由ProspectOne维护的公共库，使用的融合CDN技术，由Cloudflare、Fastly、StackPath、QUANTIL等CDN供应商提供了全球超过750个CDN节点，完全免费和无限制。最重要的是，jsDelivr在中国大陆也拥有超过数百个节点，因为jsDelivr拥有正规的ICP备案，解决了中国大陆的访问速度优化，实现真正的全球极速低延迟体验。

使用方法也十分简单，将Github的图片地址链接域名改成 jsDelivr 的 CDN 域名即可。格式如下：

https://cdn.jsdelivr.net/gh/Github用户名/仓库名@发布的版本号/图片路径

因为上传的是图片，无需发布版本，图片路径也可以在PicGo里设置，以上格式可精简为：

https://cdn.jsdelivr.net/gh/Github用户名/仓库名

将该路径改成自己的信息，填写至PicGo的自定义域名里，下次使用PicGo上传图片时，就可以直接获取到加速后的图片地址。

至此，有关图床的操作全部完成，可以安心码文章了。😀
